# x0tta6bl4 Incident Response Runbooks

Этот документ содержит пошаговые инструкции по устранению критических сбоев, обнаруженных системой мониторинга.

## 1. Alert: `DatabaseCircuitBreakerOpen` (Severity: Critical)

**Описание:** База данных (SQLite/Postgres) недоступна или отвечает слишком медленно. Предохранитель открыт для защиты ресурсов.

**Действия:**
1. **Проверить логи приложения:** Искать `DB Error caught by Circuit Breaker`.
2. **Проверить наличие свободного места на диске:** `df -h`. (Для SQLite это частая причина).
3. **Проверить права доступа к файлу БД:** `ls -l x0tta6bl4_enterprise.db`.
4. **Принудительный сброс (если БД восстановилась, но CB еще OPEN):**
   * Перезапустить сервис API.
5. **Проверить состояние БД через CLI:** `sqlite3 x0tta6bl4_enterprise.db "SELECT 1;"`.

---

## 2. Alert: `HighLatencyP95` (Severity: Warning)

**Описание:** Задержка ответов API превышает 1 секунду.

**Действия:**
1. **Идентифицировать эндпоинт:** Посмотреть лейбл `endpoint` в алерте.
2. **Проверить нагрузку на CPU/RAM:** `top` или `htop`.
3. **Проверить трейсы (OpenTelemetry):** Искать долгие спаны в Jaeger/Grafana по `correlation_id` из логов.
4. **Проверить блокировки БД:** Если это эндпоинт Marketplace, проверить, не заблокирована ли SQLite долгим процессом миграции или очистки.

---

## 3. MaaS Token Bridge RPC Failure (Resilience Triggered)

**Описание:** В логах видны ошибки `Blockchain RPC Breaker is OPEN`.

**Действия:**
1. **Проверить доступность RPC провайдера:** Попробовать `curl -X POST -H "Content-Type: application/json" --data '{"jsonrpc":"2.0","method":"eth_blockNumber","params":[],"id":1}' <RPC_URL>`.
2. **Обновить RPC_URL:** Если текущий провайдер (например, Base Sepolia) лежит, обновить URL в `.env` и перезапустить сервис.
3. **Проверить баланс газа:** Убедиться, что у аккаунта моста достаточно ETH для транзакций.

---

## 4. Alert: `LowAvailability` (Severity: Critical)

**Описание:** API возвращает ошибки в >1% случаев.

**Действия:**
1. **Глобальный поиск 5xx ошибок в логах:** `grep '"status": 5' app.log`.
2. **Проверить Rate Limiting:** Не заблокированы ли легитимные пользователи (ошибки 429).
3. **Проверить целостность PQC ключей:** Если ошибки в `/vpn/config`, проверить доступность `liboqs`.
