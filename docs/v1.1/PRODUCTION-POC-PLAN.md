# Production POC Plan (v1.1)

Этот документ определяет дорожную карту перехода от валидационного POC (симуляции) к физической интеграции Federated ML и 5G Edge.

## 1. Область применения (Scope)
Интеграция распределенного обучения (FedML) и граничных вычислений (MEC) в частных сетях 5G с использованием LoRaWAN в качестве резервного канала связи.

## 2. Фазы интеграции

### Фаза 1: Лабораторная проводка (Scaffold Wiring)
- **Цель:** Заменить программные заглушки (Simulated) на интерфейсные вызовы к реальным библиотекам в контролируемой среде.
- **Изменения статуса:**
  - 5G UPF Handoff: SIMULATED -> READY FOR LIVE VALIDATION
  - LoRa Telemetry: SIMULATED -> READY FOR LIVE VALIDATION

### Фаза 2: Полевые испытания (Pilot Deployment)
- **Цель:** Развертывание на 3-х физических кластерах с реальными шлюзами SX1303 и Open5GS.
- **Изменения статуса:**
  - Model Convergence: READY FOR LIVE VALIDATION -> VERIFIED
  - eBPF QoS Marking: NOT VERIFIED -> VERIFIED (при наличии eBPF-драйвера)

### Фаза 3: Коммерческая аттестация
- **Цель:** Подтверждение SLA и аудит безопасности PQC-слоя.
- **Изменения статуса:**
  - Fairness (Gini Index): NOT VERIFIED -> VERIFIED

## 3. Контрольные точки верификации
- [ ] Успешная агрегация градиентов от 3-х физически удаленных кластеров.
- [ ] Время переключения UPF-сессии < 50 мс в реальной сети.
- [ ] Дистанция LoRa-канала > 15 км с подтвержденным RSSI.
- [ ] Корректное списание X0T токенов за выделение QoS-слайсов.

## 4. Риски
- **Производительность PQC:** Высокие накладные расходы на подписи Dilithium могут увеличить задержку.
- **Шум DP-SGD:** Слишком высокий уровень шума для обеспечения приватности может разрушить сходимость модели.
