# LoRaWAN SX1303 Hardware Verification Guide

Этот документ описывает процедуру перевода LoRaWAN Backhaul в статус **VERIFIED** на реальном оборудовании.

## 1. Требования
- Концентратор на базе чипа **Semtech SX1303** (например, RAK5146).
- Драйвер `libloragw` версии 2.1.0+.
- Поддержка SPI или USB (через FTDI).

## 2. Процедура сбора доказательств (Evidence Collection)
Чтобы подтвердить работоспособность, необходимо выполнить следующие шаги:

### Шаг A: Инициализация HAL
Запустите тестовую утилиту Semtech для проверки чипа:
```bash
./util_chip_id -d /dev/spidev0.0
```
**Критерий успеха:** Получен корректный Chip ID. Запишите вывод в `hardware_audit.log`.

### Шаг B: Тест чувствительности (RSSI/SNR)
Разместите передатчик на расстоянии > 15 км и выполните замер через `DecideRoute`.
```bash
go test -v ./backhaul/lora/... -run TestSX1303RealMetrics
```
**Критерий успеха:** Система должна выбрать `BackhaulType: LoRa` при RSSI > -118dBm и SNR > -12dB.

## 3. Матрица продвижения (Promotion)
| Текущий статус | Действие | Новый статус |
| :--- | :--- | :--- |
| **SIMULATED** | Успешная сборка с `libloragw` | **READY FOR LIVE VALIDATION** |
| **READY FOR LIVE** | Полевой тест на 15 км | **VERIFIED** |

## 4. Ограничения
Без физического концентратора любые метрики `lora_last_packet_rssi_dbm` считаются **имитацией** и должны иметь префикс `_simulated` в дашбордах.
