## Суть продукта

Программный комплекс для защищённого обмена данными в распределённых сетях — там, где нет центрального сервера, а устройства связываются напрямую друг с другом. Основное применение — каналы управления беспилотными летательными аппаратами (БПЛА) и промышленные распределённые системы.

## Чем уникален

Стандартные алгоритмы шифрования станут уязвимы, когда появятся квантовые компьютеры. Продукт использует **постквантовую криптографию** — алгоритмы, устойчивые к взлому даже квантовым компьютером. Это общепризнанные мировые эталоны стойкости (стандарты FIPS 203/204/205).

Для БПЛА это особенно важно: канал управления — самая критичная точка. Если его взломают, аппарат можно угнать или посадить. Предлагаемое решение защищает канал управления шифрованием, которое не взламывается известными методами.

## Ключевые характеристики

- Защита от перехвата и подмены команд на канале управления
- Устойчивость к перспективным квантовым атакам (шифрование ML-KEM-768 и подписи ML-DSA-65)
- Работа в условиях нестабильной связи (потери пакетов, обрывы)
- Отказоустойчивость: при отказе одного узла сеть продолжает работу
- Само-восстановление: MAPE-K цикл автоматически исправляет сбои

## Кому это нужно

1. **Производители БПЛА** — Геоскан, EAV, Supercam и другие: защита каналов управления от перехвата
2. **Интеграторы связи и телекоммуникационные компании** — встраивание модуля шифрования в существующие системы
3. **Предприятия промышленной автоматизации** — защита распределённых сетей управления оборудованием
4. **Разработчики защищённых сетей** (ИнфоТеКС, С-Терра) — лицензирование технологии как OEM-компонента
5. **Университеты и исследовательские лаборатории** — исследования в области постквантовой криптографии
6. **Малый бизнес Крыма** — услуги по настройке и аудиту безопасности компьютерных сетей

## Почему сейчас

После ухода западных вендоров рынок защищённых сетей в РФ освободился. Российских аналогов, использующих постквантовую криптографию, пока нет. Продукт может занять эту нишу.

## Технологический стек

| Технология | Назначение |
|:-----------|:-----------|
| шифрование ML-KEM-768 (стандарт FIPS 203) | Постквантовое согласование ключей |
| электронная подпись ML-DSA-65 (стандарт FIPS 204) | Постквантовая подпись команд |
| подпись SLH-DSA / SPHINCS+ (стандарт FIPS 205) | Долгосрочная архивная подпись |
| обработка сетевых пакетов на уровне ядра (eBPF/XDP) | Обработка пакетов на уровне ядра Linux |
| SPIFFE/SPIRE | Межпроцессная аутентификация |
| MAPE-K | Само-восстановление сети |
| FastAPI | REST API для управления |
| Docker/Kubernetes | Деплой и оркестрация |
