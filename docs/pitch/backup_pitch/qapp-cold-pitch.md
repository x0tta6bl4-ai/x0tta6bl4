## Тема: Интеграция PQC SDK в реальный eBPF/MAVLink транспорт

Здравствуйте, коллеги из QApp!

Меня зовут x0tta6bl4. Я разрабатываю открытую Zero-Trust mesh-платформу с eBPF-датаплейном и постквантовой криптографией (проект x0tta6bl4, Apache-2.0, v0.2.0-rc1, 925K+ LOC, 1312 тестов).

Недавно я завершил спецификацию **x0tMQ** — расширения протокола MAVLink v2 для прозрачной ретрансляции постквантовой аутентификации через legacy-оборудование. Это IETF-style Internet-Draft, реализующий:

- **ML-KEM-768** (FIPS 203) — защищённое согласование сессионных ключей
- **ML-DSA-65** (FIPS 204) — подпись критических команд управления
- **HMAC-SHA3-256** — лёгкая аутентификация телеметрии (1.1 μs/pkt)

Ключевая инновация — **x0CHUNK**: фрагментация PQ-подписей в 245-байтные кадры, которые проходят через любые стандартные MAVLink-ретрансляторы без модификации. Relay-striping attack mitigation — на уровне протокола.

### Бенчмарки (ARM64 Neoverse-N2, liboqs):

| Операция | Время |
|----------|-------|
| ML-KEM-768 encapsulation | 241 μs |
| ML-DSA-65 sign (256 B) | 509 μs |
