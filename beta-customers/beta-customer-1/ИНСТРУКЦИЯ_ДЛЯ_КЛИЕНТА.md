# Инструкция для первого beta-клиента

**Дата:** 8 января 2026  
**Версия:** x0tta6bl4 v3.4.0-fixed2

---

## Что это

x0tta6bl4 — это mesh-сеть с защитой от квантовых компьютеров. Сейчас мы в beta, и ты помогаешь нам проверить, как всё работает в реальных условиях.

---

## Как начать

### 1. Проверь, что сервис работает

```bash
curl http://x0tta6bl4-staging.x0tta6bl4-staging.svc.cluster.local:8080/health
```

Должен вернуться ответ со статусом "ok". Если нет — напиши нам.

### 2. Посмотри метрики

```bash
curl http://x0tta6bl4-staging.x0tta6bl4-staging.svc.cluster.local:8080/metrics
```

Здесь видно, как работает система: сколько запросов, какая задержка, сколько ошибок.

### 3. Твой API-токен

```
640fe5829388d2c497988a6605aa612bb8e488f8d97b49d095eb501345375371
```

Сохрани его. Он нужен для доступа к API. Не публикуй его в открытом доступе.

## Что тестировать

### Основное

1. **Подключение к сети.** Запусти несколько узлов и проверь, что они видят друг друга.
2. **PQC-шифрование.** Проверь, что handshake работает (ML-KEM-768, ML-DSA-65).
3. **Самоисцеление.** Останови один pod и посмотри, как система восстановится.
4. **Производительность.** Посмотри метрики: задержка, пропускная способность, использование ресурсов.

### Дополнительно

- Anomaly detection (GraphSAGE)
- Federated Learning (если доступно)
- eBPF observability (если доступно)

## Если что-то не работает

### Сервис не отвечает

Проверь статус pods:
```bash
kubectl get pods -n x0tta6bl4-staging
```

Если pods не Running — напиши нам.

### Высокая задержка

Посмотри метрики в Prometheus. Если нужно — дай нам знать, и мы разберёмся.

### Проблемы с аутентификацией

Проверь токен. Если не помогает — напиши, поможем.

- Это не production. Могут быть баги.
- Масштаб ограничен. Не стоит нагружать систему слишком сильно.
