# NFT Badge Design Brief

## Overview
Create 10 NFT badge designs representing early supporter roles in the x0tta6bl4 mesh & resilience ecosystem.

## Badge Levels
1. Node Supporter (Base) — Entry participation.
2. Mesh Architect — System design contributor.
3. Resilience Guardian — High-tier governance & audits.
4. Policy Sentinel — Focus on OPA / Zero-Trust rules.
5. Quantum Pioneer — Post-quantum crypto advocacy.
6. Accessibility Champion — Inclusion & localization.
7. Routing Alchemist — ML / GNN optimization contributor.
8. Disaster Response Ally — Field deployment supporter.
9. DAO Steward — Governance process enhancer.
10. Signal Bearer — Ambassador spreading awareness.

## Visual Direction
- Style: Minimal techno-organic (mesh nodes + soft neon links).
- Palette: Deep midnight (#0d1117), Electric cyan (#3ddfff), Signal amber (#ffb347) for warning tiers.
- Shape Language: Interlocking hex cells (mesh topology) + identity glyph overlays.
- Motion (optional): Subtle pulsing link path (GIF/MP4 variant for marketing).

## Layers (Suggested)
- Base: Gradient field (tier color).
- Mesh Pattern: Semi-transparent node lattice.
- Emblem: Unique glyph per role (e.g., shield, brain network, accessibility icon, quantum key).
- Text: Short code (NODE / ARCH / GUARD / POL / QNT / ACC / RTG / DRP / DAO / SIG).
- Rarity Mark: Dot clusters (tier increments).

## Deliverables
- 10 static PNG (1024x1024)
- 10 SVG scalable versions
- Optional animated (5 badges) MP4/WebM
- Metadata JSON template (name, description, attributes)

## Attribute Suggestions
- Tier Level (1–3)
- Governance Weight (1x, 1.5x, 2x)
- Domain (security, routing, accessibility, governance)
- Rarity (% encoded visually)

## Timeline
- Sketch concepts: Day 1
- Feedback & revisions: Day 2
- Final export + metadata: Day 3

## Notes
Badges should feel part of one visual system; emphasize clarity at small size (Discord / wallet preview). Avoid licensed fonts; prefer Open Source (e.g., Inter or JetBrains Mono).
