# Полное руководство по созданию скиллов Claude

Источник: The Complete Guide to Building Skills for Claude (Anthropic, 2026)

## Что такое скилл

Скилл — это папка с инструкциями, которая учит Claude выполнять специфические задачи. Вместо повторного объяснения предпочтений в каждом разговоре, скиллы позволяют научить Claude один раз.

## Структура скилла

```
skill-name/
├── SKILL.md         # Обязательно - главный файл
├── scripts/         # Опционально - исполняемый код (Python, Bash)
├── references/      # Опционально - документация для загрузки по требованию
└── assets/          # Опционально - шаблоны, шрифты, иконки
```

## Принципы дизайна

### Progressive Disclosure (Прогрессивное раскрытие)

Три уровня загрузки:
1. **Первый уровень (YAML frontmatter)** — всегда в system prompt (~100 токенов)
2. **Второй уровень (тело SKILL.md)** — загружается когда скилл релевантен
3. **Третий уровень (связанные файлы)** — загружаются по необходимости

### Composability (Композируемость)

Claude может загружать несколько скиллов одновременно. Скилл должен работать вместе с другими.

### Portability (Портативность)

Скиллы работают идентично в Claude.ai, Claude Code и API.

## YAML Frontmatter

### Минимальный формат

```yaml
---
name: your-skill-name
description: What it does. Use when user asks to [specific phrases].
---
```

### Требования к полям

**name (обязательно):**
- Только kebab-case
- Без пробелов и заглавных букв
- Должно совпадать с именем папки

**description (обязательно):**
- ДОЛЖНО содержать ОБА:
  - Что делает скилл
  - Когда его использовать (триггеры)
- Максимум 1024 символа
- Без XML тегов (< или >)
- Включать конкретные фразы пользователей
- Упоминать типы файлов если релевантно

**license (опционально):**
- MIT, Apache-2.0 и т.д.

**compatibility (опционально):**
- 1-500 символов
- Требования к окружению

**metadata (опционально):**
- author, version, mcp-server

### Запрещено в frontmatter

- XML угловые скобки (< >)
- Слова "claude" или "anthropic" в name

## Примеры хороших description

```yaml
# Хорошо - конкретно и actionable
description: Analyzes Figma design files and generates developer handoff documentation. Use when user uploads .fig files, asks for "design specs", "component documentation", or "design-to-code handoff".

# Хорошо - включает триггерные фразы
description: Manages Linear project workflows including sprint planning, task creation, and status tracking. Use when user mentions "sprint", "Linear tasks", "project planning", or asks to "create tickets".
```

## Примеры плохих description

```yaml
# Слишком размыто
description: Helps with projects.

# Нет триггеров
description: Creates sophisticated multi-page documentation systems.

# Слишком техническое, нет триггеров пользователя
description: Implements the Project entity model with hierarchical relationships.
```

## Категории скиллов

### Category 1: Document & Asset Creation

Для создания документов, презентаций, кода, дизайна.

Ключевые техники:
- Встроенные style guides
- Шаблоны для консистентного вывода
- Чеклисты качества перед финализацией

### Category 2: Workflow Automation

Для многошаговых процессов с последовательной методологией.

Ключевые техники:
- Пошаговый workflow с gates валидации
- Шаблоны для типичных структур
- Встроенные предложения по улучшению
- Циклы итеративного улучшения

### Category 3: MCP Enhancement

Для улучшения инструментов MCP сервера.

Ключевые техники:
- Координация множественных вызовов MCP
- Встроенная доменная экспертиза
- Контекст для пользователей
- Обработка типичных ошибок MCP

## Критерии успеха

### Количественные метрики
- Скилл триггерится на 90% релевантных запросов
- Завершает workflow за X вызовов инструментов
- 0 неудачных API вызовов

### Качественные метрики
- Пользователям не нужно подсказывать следующие шаги
- Workflows завершаются без коррекции
- Консистентные результаты между сессиями

## Тестирование

### Triggering tests
- ✅ Триггерится на очевидных задачах
- ✅ Триггерится на перефразированных запросах
- ❌ Не триггерится на несвязанных темах

### Functional tests
- Генерируются корректные выводы
- API вызовы успешны
- Обработка ошибок работает
- Edge cases покрыты

## Распространение

### Текущая модель (январь 2026)

1. Скачать папку скилла
2. Заархивировать (если нужно)
3. Загрузить в Claude.ai через Settings > Capabilities > Skills
4. Или поместить в директорию скиллов Claude Code

## Что НЕ включать в скилл

- README.md
- INSTALLATION_GUIDE.md
- QUICK_REFERENCE.md
- CHANGELOG.md

Скилл должен содержать только информацию для AI агента. Не включать вспомогательный контекст, процедуры тестирования, документацию для пользователей.

## Рекомендации по размеру

- SKILL.md < 500 строк
- SKILL.md < 5000 слов
- Детальную документацию выносить в references/
- Если файлы > 10k слов, включать grep patterns в SKILL.md
