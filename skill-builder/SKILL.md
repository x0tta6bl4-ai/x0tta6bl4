# Skill Builder

Скилл для создания других скиллов Claude с проверкой по документации и актуальным best practices.

## Обязательный рабочий процесс

**КРИТИЧЕСКИ ВАЖНО**: Выполнять шаги строго по порядку, ничего не пропуская.

### Шаг 1: Сбор требований

Узнать у пользователя:
1. Какую задачу должен решать скилл?
2. Какие триггерные фразы должны его активировать?
3. Нужны ли скрипты, референсы или ассеты?
4. Есть ли примеры использования?

Не задавать больше 2-3 вопросов за раз.

### Шаг 2: Проверка документации

**Обязательно** прочитать `references/complete-guide.md` для:
- Структуры SKILL.md
- Правил именования
- Требований к frontmatter
- Паттернов progressive disclosure

### Шаг 3: Поиск актуальных практик

**Обязательно** выполнить веб-поиск:
```
Claude skills SKILL.md best practices [текущий год]
```

Искать:
- Обновления формата
- Новые рекомендации Anthropic
- Примеры от сообщества

### Шаг 4: Валидация плана

Перед созданием файлов проверить:

**Чеклист структуры:**
- [ ] Имя в kebab-case (без пробелов, без заглавных)
- [ ] description содержит ЧТО делает И КОГДА использовать
- [ ] Нет XML тегов (< >) в frontmatter
- [ ] Инструкции конкретные и actionable
- [ ] SKILL.md < 500 строк

**Чеклист контента:**
- [ ] Добавлена только информация, которую Claude не знает
- [ ] Примеры вместо длинных объяснений
- [ ] Детали вынесены в references/
- [ ] Нет README.md, CHANGELOG.md и подобных файлов

### Шаг 5: Создание скилла

Структура папки:
```
skill-name/
├── SKILL.md           # Обязательно
├── scripts/           # Опционально - исполняемый код
├── references/        # Опционально - документация
└── assets/            # Опционально - шаблоны, иконки
```

**Формат SKILL.md:**
```yaml
---
name: skill-name-kebab-case
description: Что делает скилл. Использовать когда [конкретные триггеры и фразы].
---

# Название скилла

## Инструкции

[Пошаговые действия в императивной форме]

## Примеры

[Конкретные примеры использования]

## Обработка ошибок

[Типичные проблемы и решения]
```

### Шаг 6: Финальная валидация

Запустить `scripts/validate_skill.py` для проверки:
- Корректность YAML frontmatter
- Наличие обязательных полей
- Соответствие naming conventions
- Отсутствие запрещённых элементов

### Шаг 7: Упаковка

Создать .zip архив с папкой скилла для загрузки в Claude.ai.

## Частые ошибки

**Скилл не триггерится:**
- Слишком общий description
- Нет конкретных триггерных фраз
- Нет ключевых терминов

**Скилл триггерится слишком часто:**
- Добавить негативные триггеры
- Уточнить scope
- Быть более специфичным

**Инструкции не выполняются:**
- Сделать инструкции короче
- Вынести детали в references/
- Использовать ## КРИТИЧНО для важного
