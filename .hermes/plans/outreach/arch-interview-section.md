# 15-минутная архитектурная секция для собеседования

## Структура (15 мин)

### 0:00-2:00 — Проблема (зацепка)

> *«AI-агенты пишут код в 10× быстрее человека. Bottleneck сместился с написания кода на его верификацию. Как проверить, что AI-агент не сломал логику, если тесты тоже написал AI?»*

### 2:00-7:00 — Решение: Agentic QA Engineering (мой флагман)

**Проблема 1: Доверять «готово» нельзя**
- programmatic checks (pytest exit 0, py_compile, import src)
- judge verdict (вторая LLM с structured JSON: pass/revise + blocking_issues + confidence)
- human gate (только в точках высокого рычага)

**Проблема 2: Цикл может уйти в бесконечность**
- max_iterations, max_revisions, no_progress detector
- token budget advisory

**Проблема 3: Модель забывает предыдущий запуск**
- Внешнее состояние: run-log.md, state.json, VERSION — всё в репозитории

**Результат:** автономный CVE monitor — 20 CVEs за день, 0 ошибок, 0 ручных действий.

### 7:00-10:00 — Loop Engineering (как это проектируется)

**Процесс из 7 этапов (портирован с Looper/ksimback):**
1. Goal — формулирую проверяемую цель
2. Verification — типизирую критерии (programmatic / judge / human)
3. Host — выбираю модель-исполнителя
4. Council — подключаю модель-судью через delegate_task
5. Gates — ставлю лимиты итераций и бюджеты
6. Confirmation — ASCII-схема перед запуском
7. Emit — сохраняю план, устанавливаю todo

**Живой пример:** cronjob на 6:00 UTC — сам проверяет dependabot, чинит CVE, пушит.

### 10:00-13:00 — Архитектура системы (подтверждение компетенций)

**11 слоёв (2 мин):**
1. eBPF/XDP — ядро (142k PPS)
2. PQC ML-KEM+ML-DSA — криптография (50ms handshake)
3. Ghost Transport — обход DPI
4. SPIRE/SPIFFE mTLS — identity
5. Mesh routing — batman-adv
6. MAPE-K — самовосстановление (MTTR 3 мин)
7. MaaS API — monetization
8. DAO smart contracts — governance

**Ключевое:** все 11 слоёв написаны AI-агентами. Моя роль — архитектура, промпты, верификация.

### 13:00-15:00 — Почему это важно сейчас (закрытие)

> *«По данным анализа рынка (стрим Острикова, SWE-bench, The Register article on Loop Engineering): код становится бесплатным, bottleneck — верификация. Компании, которые не построят Agentic QA, утонут в AI-generated коде.»*

**Что я ищу:** возможность спроектировать систему верификации AI-агентов под ваши задачи. Не писать код — проектировать процессы.
